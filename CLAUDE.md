# CLAUDE.md - LSTM 时序预测项目开发准则

## 项目目标
基于 PyTorch 从 0 到 1 搭建并训练 LSTM 时序模型，当前用于便利店营业额预测，未来将扩展到股票量化领域。

## 核心原则

### 1. 文档优先原则 (Docs-First)
每次完成以下操作后，**必须**主动在 `docs/learning_journal.md` 中追加学习日志：
- 添加新功能
- 修改代码
- 解决 Bug 并得到有效结论

### 2. 日志格式强制要求
每条记录必须包含以下5个字段：

```
[日期时间]
[当前任务/遇到的问题]
[分析与思考过程]
[最终解决方案/核心代码片段]
[核心知识点总结]
```

### 3. 环境规范
本项目严格使用 **Anaconda** 管理依赖。代码执行必须在 `lstm_sales_env` 环境下进行。任何时候引入了新的第三方库，**必须**同步更新到 `environment.yml` 文件中，保证项目的可一键复现性。

因网络环境原因，`environment.yml` 已配置清华/阿里云镜像源。后续新增包时，Conda 包默认走清华源；若必须使用 Pip 安装，需带上 `-i https://mirrors.aliyun.com/pypi/simple/` 参数。

### 4. 编码规范
- **高可扩展性**：代码需支持未来扩展到金融股票数据
- **参数解耦**：超参数、数据路径、模型结构等需易于配置
- **模块化设计**：数据处理、模型定义、训练逻辑、推理逻辑需分离
- **类型提示**：函数参数和返回值需标注类型
- **日志记录**：使用 `logging` 模块记录关键信息

## 目录结构规范
```
├── data/           # 原始xlsx和处理后的数据（真实数据不提交）
├── models/         # 训练好的模型权重 .pth
├── scripts/        # 训练、推理、数据预处理等独立脚本
├── docs/           # 学习日志
├── environment.yml # Conda 环境配置文件
├── CLAUDE.md       # 本文件
└── README.md       # 项目说明
```

## 模型设计规范
- 模型类需继承 `torch.nn.Module`
- 必须实现 `forward()` 方法
- 训练循环需支持早停 (Early Stopping)
- 支持 GPU/CPU 自适应

## 数据规范
- 时间序列数据需保留时间戳索引
- 数据标准化使用 `sklearn.preprocessing` 或自定义
- 滑窗大小 (window_size) 需作为超参数可配置
