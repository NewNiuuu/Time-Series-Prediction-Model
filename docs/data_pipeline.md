# 数据流管道文档

> 本文档从**数据流**的角度，梳理从最初的原始数据开始，如何一步步经过处理、建模，最终得到预测结果的完整路径。

## 二、所有落地文件汇总

| 文件路径 | 类型 | 作用 | 生成者 |
|---------|------|------|--------|
| `data/processed_daily_sales.csv` | CSV | 每日归一化营业额序列 | `data_processor.py` |
| `models/scaler.pkl` | pickle | 归一化器（预测时必须加载） | `data_processor.py` |
| `models/lstm_baseline.pth` | PyTorch state_dict | 基准模型权重 | `train.py` |
| `models/lstm_finetuned.pth` | PyTorch state_dict | 增量微调后模型权重 | `inference.py` |
| `data/simulated_new_data.csv` | CSV | 模拟的新数据（测试用） | `inference.py` (if __main__) |

## 三、数据维度变换全览

```
原始输入
  20251001-20251231.xlsx    →    约 8000 条逐笔交易记录
  20260101-20260401.xlsx    →    约 3000 条逐笔交易记录
                                      │
                              data_processor.py
                                      ↓
                            11528 条逐笔交易记录
                            (date, type, amount)
                                      │
                            data_processor.py 重采样
                                      ↓
                            183 天的每日营业额序列
                            [248.00 ~ 8092.71 元]
                                      │
                            data_processor.py 归一化
                                      ↓
                            183 个 [0, 1] 之间的浮点数
                                      │
                            dataset.py 滑动窗口切分
                                      ↓
                            169 个样本
                            X: (14, 1)  y: (1,)
                                      │
                            dataset.py DataLoader 批次化
                                      ↓
                            batch_X: (16, 14, 1)
                            batch_y: (16, 1)
                                      │
                            train.py 训练
                                      ↓
                            模型权重 lstm_baseline.pth
                            scaler.pkl（不变）
                                      │
                            inference.py 推理
                                      ↓
                            单日预测值（归一化尺度）→ 反归一化 → 2655.41 元
```

## 四、为什么每个变换步骤都是必需的

| 变换步骤 | 如果没有会怎样 |
|---------|-------------|
| 逐行搜索定位真实表头 | 把说明文本当数据，解析完全错误 |
| 列名归一化 | 2025年文件的"金额"列和2026年的"金额(元)"列无法合并 |
| 只保留"收入"过滤 | 支出记录会被当成收入，营业额数据偏高 |
| 按天重采样 | 时间步长不一致，LSTM 无法处理不等长序列 |
| 缺失日期补0 | 滑窗时产生错误对应（把不同日期的数据当成连续的） |
| MinMaxScaler [0,1] | 模型收敛极慢，不同量纲导致梯度方向扭曲 |
| 滑动窗口 seq_length=14 | 无法构建监督学习的 (X, y) 样本对 |
| shuffle=False | 打乱后时间依赖关系被破坏，模型学不到顺序规律 |
| 早停机制 | Epoch 2 后验证 loss 上升，持续训练严重过拟合 |
| 保存 scaler.pkl | 推理时无法将归一化值还原为"元"，无法给业务提供可理解的数字 |
